## High-level view

The auth system is functional end-to-end for a mock app: Zustand auth store with MMKV persistence, login screen passes a mock token, and `AuthNavigationGuard` in the root layout handles redirects. However, the profile screen retains a fallback `login()` call that conflicts with this architecture â€” it silently re-authenticates when `user` is null, masking state inconsistencies the guard should catch.

The `fill` style prop on the Feather heart icon is dead code. Feather icons render as font glyphs via `Text` components; they don't support SVG fill. The color prop is the actual differentiator and works correctly, but the no-op style and its comment are misleading.

Zustand's `persist` middleware rehydrates asynchronously from MMKV. The auth guard doesn't wait for hydration, so on cold start `isAuthenticated` is momentarily `false` even when a session exists. This triggers a redirect to login, then rehydration fires and redirects back. Cosmetic for a demo, but would need a hydration gate in production.

## Profile screen residual force-login

In `app/(tabs)/profile.tsx`:

```tsx
useEffect(() => {
  if (!user) {
    login({
      id: currentUser.id,
      username: currentUser.username,
      displayName: currentUser.displayName,
      avatar: currentUser.avatar,
      bio: currentUser.bio,
    });
  }
}, []);
```

With the auth guard now in place, a user cannot reach the profile screen without being authenticated. If `user` is somehow null despite `isAuthenticated` being true (a state inconsistency), silently fabricating a session hides the bug. This call should be removed â€” the guard guarantees auth state is populated before this screen renders.

## Feather icon fill style no-op

In `PostCard.tsx`, the heart icon applies:

```tsx
style={post.isLiked ? { fill: theme.colors.accent.primary } as TextStyle : undefined}
```

Feather icons from `@expo/vector-icons` render via a custom font, not SVG elements. The `fill` property has no effect on React Native `Text` components. The comment stating "The filled visual is achieved by rendering the icon in the accent color" contradicts the style prop â€” if color is the mechanism (and it is), the `fill` style is dead code that suggests a misunderstanding of how icon fonts work.

## Auth guard timing

The `AuthNavigationGuard` in `app/_layout.tsx`:

```tsx
useEffect(() => {
  const inAuthGroup = segments[0] === '(auth)';
  if (!isAuthenticated && !inAuthGroup) {
    router.replace('/(auth)/login');
  } else if (isAuthenticated && inAuthGroup) {
    router.replace('/(tabs)');
  }
}, [isAuthenticated, segments]);
```

Because Zustand's `persist` middleware rehydrates asynchronously from MMKV, `isAuthenticated` is `false` on the first render even when a valid session exists in storage. This causes a redirect to login, then rehydration fires, `isAuthenticated` flips to `true`, and the guard redirects back to tabs â€” a visible flash. The fix is to check `useAuthStore.persist.hasHydrated()` and render nothing (or a splash) until hydration completes.
